# WebEye 项目技术栈与功能分析

## 技术栈

### 核心技术
- **Spring Boot 3.5.13**：作为应用程序的基础框架，提供自动配置和依赖管理
- **Java 17**：主要开发语言，提供现代Java特性支持
- **Spring Security 6**：实现用户认证和授权，保护应用程序安全
- **Spring Data JPA**：简化数据库操作，提供对象关系映射
- **MySQL**：关系型数据库，存储用户信息和扫描任务数据
- **Thymeleaf**：服务器端HTML模板引擎，用于构建动态Web界面
- **Lombok**：减少样板代码，提高开发效率
- **异步处理**：使用`@EnableAsync`和`@Async`注解实现后台扫描任务

### 项目结构
```
src/main/java/com/webeye/
├── config/            # 配置类
├── controller/        # 控制器
├── dto/               # 数据传输对象
├── entity/            # 实体类
├── exception/         # 异常处理
├── repository/        # 数据访问层
├── security/          # 安全配置
├── service/           # 业务逻辑层
└── WebeyeApplication.java  # 应用主类
```

## 功能模块

### 1. 用户认证与管理模块
- **用户注册**：支持新用户注册，包括用户名、密码和邮箱验证
- **用户登录**：基于Spring Security的认证机制
- **权限控制**：基于角色的访问控制，支持不同用户权限
- **用户信息管理**：获取和展示用户信息

### 2. Web安全扫描模块
- **扫描任务管理**：创建、查看和管理扫描任务
- **异步扫描**：后台执行扫描任务，不阻塞用户操作
- **漏洞检测**：模拟检测常见Web漏洞
- **漏洞等级评估**：将漏洞分为高、中、低三个等级
- **扫描历史记录**：保存和展示历史扫描结果

### 3. 界面模块
- **登录页面**：用户认证入口
- **仪表盘**：扫描任务管理和提交
- **扫描历史**：查看历史扫描记录
- **扫描详情**：展示漏洞详细信息
- **中文本地化**：所有界面和提示信息均为中文，确保用户友好

## 核心实现

### 数据模型
- **User**：用户实体，实现`UserDetails`接口，支持Spring Security认证
- **ScanTask**：扫描任务实体，包含目标URL、状态和时间信息
- **Vulnerability**：漏洞实体，包含类型、等级、描述和位置信息

### 关键功能实现
1. **用户认证**：使用Spring Security的表单认证，密码加密存储
2. **异步扫描**：通过`@Async`注解实现后台扫描任务
3. **漏洞检测**：模拟检测SQL注入、XSS、CSRF等常见Web漏洞
4. **异常处理**：全局异常捕获，统一返回格式
5. **数据初始化**：自动创建管理员用户和示例数据

### 安全特性
- **密码加密**：使用Spring Security的密码编码器
- **CSRF防护**：Spring Security内置CSRF保护
- **权限控制**：基于角色的访问控制
- **安全头**：Spring Security默认安全头设置

## 项目亮点
1. **完整的用户认证系统**：从注册到登录，权限控制完整
2. **异步扫描设计**：后台执行扫描任务，提升用户体验
3. **中文本地化**：全中文界面，适合国内用户
4. **模块化设计**：清晰的代码结构，易于维护和扩展
5. **安全最佳实践**：遵循Spring Security的安全最佳实践

## 技术价值
- **学习价值**：涵盖了Spring Boot、Spring Security、JPA等核心技术的综合应用
- **实用价值**：提供了一个功能完整的Web安全扫描平台原型
- **架构价值**：展示了现代Java Web应用的最佳实践架构

这个项目展示了如何使用Spring Boot生态系统构建一个功能完整的Web应用，特别强调了安全性和用户体验。它不仅是一个技术学习的范例，也是一个可以进一步扩展的实用工具。
