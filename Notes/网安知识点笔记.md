# DAY 1: 基础入门——概念名词

## 一、域名
1.  **概念**
    相当于IP地址的名字（IP地址不便记忆）。可通过**域名名称系统（DNS，Domain Name System）**将域名和IP相互映射起来。

2.  **注册**
    在第三方平台注册，例如：
    *   万网（阿里云）国内
    *   GoDaddy

3.  **二级域名/多级域名**
    **例如**：`www.baidu.com`
    *   `.com` 为一级域名（顶级域名）
    *   `baidu` 为二级域名
    *   `www` 为三级域名

4.  **域名对安全测试的意义**
    *   **① 相同二级域名的网站架构、源码、密码可能相似** → 可进行**旁站攻击**。
    *   **② 两个网站域名解析后的IP地址可能相同** → 可从**旁站进入主站**。

## 二、DNS
1.  **概念**
    **域名系统（服务）协议（Domain Name System）**，用于域名和IP地址相互转换。DNS服务器可手动设置。

2.  **本地 Host 文件与 DNS 的关系**
    *   **Host 文件路径**：`C:\Windows\System32\drivers\etc\hosts`
    *   **作用**：在查找域名解析时，先查看本地 Host 文件是否有缓存，没有则查找域名服务器。（**可被用于钓鱼攻击**）

3.  **CDN 和 DNS 的关系**
    *   **CDN**：内容分发网络/节点技术（Content Delivery Network），将网站内容缓存到全球各地的节点，就近提供给用户，解决访问过慢问题。
    *   **关系**：DNS 是引导用户访问 CDN 的关键。当用户访问一个使用了 CDN 的网站时，DNS 系统会将用户的域名解析到**最合适的 CDN 节点的 IP 地址**，而不是源服务器的 IP 地址。

| 步骤 | 说明 |
| :--- | :--- |
| 1 | 用户输入域名访问网站 |
| 2 | DNS 解析域名，返回最接近用户的 CDN 节点 IP |
| 3 | 用户直接访问 CDN 节点获取缓存内容 |

4.  **常见 DNS 攻击**
| 攻击类型 | 说明 |
| :--- | :--- |
| **DNS 劫持** | 攻击服务器（地区），导致 IP 解析不正常 |
| **DNS 跳转** | 诱导用户跳转至钓鱼网站 |

# Day2：基础入门-数据包拓展

## 一、HTTP/HTTPS数据包基础

### 1. HTTP/HTTPS区别
HTTPS相比HTTP，多了一道SSL/TLS加密过程，使得数据传输更加安全。

### 2. HTTP简要通信流程
1.  **建立连接**：浏览器建立与Web服务器之间的TCP连接。
2.  **发送请求**：浏览器将请求数据打包（生成请求数据包）并发送到Web服务器。
3.  **返回响应**：Web服务器将处理结果打包（生成响应数据包）并发送给浏览器。
4.  **关闭连接**：Web服务器关闭连接（在HTTP/1.0中；HTTP/1.1默认启用Keep-Alive可复用连接）。

### 3. HTTP请求数据包格式
一个HTTP请求数据包主要由四部分组成：**请求行、请求头、空行、请求体**。

*   **请求行**：由请求方法、请求URL和协议版本三部分构成，以空格分隔。
*   **请求头**：由一系列“关键字/值”对组成，每行一对，关键字和值之间用冒号分隔。请求头用于通知服务器关于客户端的环境、能力和要求。
*   **空行**：请求头之后必须有一个空行，用于通知服务器请求头部分已结束。
*   **请求体**：包含需要发送给服务器的数据，通常在POST、PUT等方法中使用。

## 二、HTTP请求方法
HTTP协议定义了8种可能的请求方法：
*   **GET**：向指定的资源发出“显示”请求，用于获取数据。
*   **HEAD**：与GET方法相同，但服务器只返回状态行和头信息，不返回实际内容。
*   **POST**：向指定资源提交数据（例如提交表单或上传文件）。
*   **PUT**：向指定资源位置上传其最新内容。
*   **DELETE**：请求服务器删除指定的资源。
*   **OPTIONS**：查询服务器支持的HTTP请求方法。
*   **TRACE**：回显服务器收到的请求，主要用于测试或诊断。
*   **CONNECT**：预留给能够将连接改为管道方式的代理服务器，通常用于SSL隧道。

## 三、请求头理论说明
请求头由一系列“关键字/值”对组成，每行一对，关键字和值之间用冒号分隔。以下是常见的请求头字段及其理论说明：

*   **Host**：指定请求要发送到的服务器主机名和端口号。
*   **Accept**：声明客户端能够接收的媒体类型（MIME类型）。
*   **User-Agent**：包含发出请求的用户代理（通常是浏览器）的应用程序类型、操作系统、软件厂商和版本信息。
*   **Accept-Language**：声明客户端可理解的自然语言偏好。
*   **Connection**：控制本次连接是否在事务处理后保持打开。
*   **Cookie**：将之前服务器通过`Set-Cookie`发送的Cookie信息回传给服务器。
*   **Referer**：表示当前请求是从哪个页面链接过来的。
*   **Content-Type**：当请求方法为POST或PUT时，此头标指示请求体的媒体类型。
*   **Accept-Charset**：声明客户端可以接受的字符编码集。
*   **Accept-Encoding**：声明客户端能够理解的内容编码方式（如gzip, deflate），用于压缩传输。

## 四、HTTP响应数据包格式
一个HTTP响应数据包由四部分组成：**状态行、响应头、空行、响应体**。

*   **状态行**：由协议版本、状态码和状态描述三部分组成，以空格分隔。
*   **响应头**：由一系列“关键字/值”对组成，用于向客户端传递关于响应的附加信息，如服务器信息、资源属性、设置Cookie等。
*   **空行**：响应头之后有一个空行，用于分隔响应头和响应体。
*   **响应体**：包含服务器返回给客户端的实际内容，如HTML页面、图片、JSON数据等。

## 五、HTTP响应码分类
HTTP响应码用于表示请求的处理结果，分为五类：
*   **1xx（信息）**：请求已被接收，继续处理。
*   **2xx（成功）**：请求被成功接收、理解并接受。
*   **3xx（重定向）**：需要客户端采取进一步的操作才能完成请求。
*   **4xx（客户端错误）**：请求包含语法错误或无法被满足。
*   **5xx（服务器错误）**：服务器在处理请求的过程中发生了错误。
