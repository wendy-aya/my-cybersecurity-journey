## TryHackMe: 常见 HTTP 方法对照表
| 方法 | 核心作用 | 典型场景 | 特点 |
|------|----------|----------|------|
| **GET** | 从服务器**获取**资源 | 浏览网页、获取数据列表、查看图片等 | 1. 数据在URL中可见（参数附在URL后）<br>2. **幂等** - 多次请求同一资源结果相同<br>3. 只读操作，不改变服务器状态 |
| POST | 向服务器**提交**数据，通常会改变服务器状态 | 提交表单、上传文件、创建新资源（如发微博） | 数据放在请求体里，URL不可见；**非幂等**（多次提交可能产生不同结果，如重复下单） |
| PUT | 向服务器**更新或替换**一个完整的资源 | 修改用户信息、上传并覆盖一个文件 | **幂等**（多次请求结果一致，如把用户昵称改成同一个值） |
| DELETE | 从服务器**删除**资源 | 删除一篇文章、删除一个用户 | **幂等**（多次删除同一个不存在的资源，结果都是“已删除”） |
| HEAD | 和GET几乎一样，但**只返回响应头**，不返回响应体 | 检查资源是否存在、获取文件大小或更新时间 | 比GET更轻量，用于快速检查 |
| PATCH | 对服务器上的资源进行**部分更新** | 只修改用户的手机号，而不是整个用户信息 | **非幂等**（多次部分更新可能产生不同结果） |
| OPTIONS | 询问服务器支持哪些**HTTP方法** | 跨域请求前的“预检”，确保请求安全 | 用于复杂请求的前置检查 |

## TryHackMe: Linux核心命令摘要

### 基础信息与文件操作命令
这些是进行系统导航和查看文件内容必须掌握的基础命令。

| 命令 | 描述 | 关键用法/示例 |
| :--- | :--- | :--- |
| **`echo`** | 输出（回显）我们提供的任意文本。 | `echo Hello` 或 `echo "Hello Friend!"` |
| **`whoami`** | 查看当前登录的用户名。 | `whoami` |
| **`ls`** | **列出**当前目录下的文件和文件夹。 | `ls` |
| **`cd`** | **切换目录**，用于导航文件系统。 | `cd 目录名` |
| **`pwd`** | **打印工作目录**，显示当前所在的完整路径。 | `pwd` |
| **`cat`** | **连接**文件内容并输出，常用于查看文件。 | `cat 文件名` |

### 高级查找与搜索命令
这些命令用于在文件系统或文件内容中精确查找信息，是渗透测试中信息收集的关键。

| 命令 | 描述 | 关键用法/示例 |
| :--- | :--- | :--- |
| **`find`** | 在目录树中根据条件（如名称、类型）**查找文件**。 | 1. 按名查找: `find -name passwords.txt`<br>2. 通配符查找: `find -name *.txt` |
| **`grep`** | 在文件内容中**搜索**指定的字符串或模式。 | 1. 基本搜索: `grep "搜索词" 文件名`<br>2. 递归搜索: `grep -R "搜索词" 目录/` |
| **`wc`** | **字数统计**，常与 `-l` 参数连用以统计文件行数。 | `wc -l access.log` |

### Shell 操作符
这些操作符用于控制命令的执行流程、组合命令或重定向输入输出，对编写高效命令和脚本至关重要。

| 操作符 | 描述 | 作用与示例 |
| :--- | :--- | :--- |
| **`&`** | 将命令置于**后台运行**，不阻塞当前终端。 | `长时间任务 &` |
| **`&&`** | **逻辑与**。依次执行多个命令，只有前一个**成功**，后一个才会执行。 | `命令1 && 命令2` |
| **`>`** | **输出重定向**。将命令的输出**覆盖**写入到指定文件。 | `echo "内容" > 文件` |
| **`>>`** | **输出追加重定向**。将命令的输出**追加**到指定文件的末尾。 | `echo "新内容" >> 文件` |
